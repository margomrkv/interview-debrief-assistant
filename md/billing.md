Я тут понял одну серьезное архитектурное ограничение, которые с архитектурой в общем-то не связано. 

Antropic явным образом запретил использовать подписку с внешними harness-ами (типа Openclaw) с 4 апреля. Также подписку нельзя использовать с Agent SDK, с помощью которого можно написать код, который вызывает части Claude Code.

Для нас это значит, что если мы хотим написать нормальное приложение на Python, то нам нужно платить за API, и мы это будет прямо серьезный удар по бюджету на мощных моделях. 

Альтернативный вариант - это встраивать нашего агента в claude code, там есть вариант создать агентов с помощью .md файлов, агенты могут вызывать код итд итп

Но это обратная ситуация когда текст на естественном язык (промпт) вызывает код, но агенты близкие к продакшену делают наоборот (код вызывает текст на естественном языке).

Несмотря на меньшую инженерную составляющую, второй вариант может быть более устойчивым с точки зрения биллинга, так как мы будем платить только за вызовы внутри claude code. Также он проще.


| Внешний цикл | Claude создаёт промпт / Claude вызывает код       | Ваш код создаёт промпт / Код вызывает Claude            |
| ------------ | ------------------------------------------------- | ------------------------------------------------------- |
| Harness      | Claude Code (официальный)                         | Ваш (Agent SDK, shell-wrapper, свой бот)                |
| Биллинг      | Подписка                                          | API key                                                 |
| Пример       | Вы в сессии → Claude зовёт Bash/Agent/hooks//loop | await query(prompt) в вашем Python-скрипте-оркестраторе |
