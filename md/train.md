## Требования

мне надо реализовать ScoringPromptTrainer - подбор промпта так, чтобы он точно оценивал оценки на mocked interview. Как это можно сделать?                                                                     
                                                                                                                                                                                                                 
Тут будут оцениваться метрики фактической точности, фокуса ответа и ясности ответа. Каждая метрика будет оцениваться по 5 бальной шкале.                                                                     
                                                                                                                                                                                                             
Строится golden set (ручная разметка эталонных оценок по подмножеству Q&A).                                                                                                                                    
Разбиение train / test на уровне целых mock-сессий (чтобы не протекали коррелированные ответы из одного диалога). Ориентир пропорций, согласованный со спекой: около 70% сессий на train, около 30% на test    
                                                                             
                                                                                                                                                                                                             
Как указано выше, делается оптимизация таким образом, чтобы промпт оценивал максимально близко к оценкам из мок интервью. Для оценки будет использоваться метрика MAPE:                                        
так как она относительная - то есть интерпретируемая                                                                                                                                                           
все вопросы равнозначные (WAPE не требуется)
