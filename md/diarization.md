# Диаризация

**Q: А верно ли, что на YouTube нет диаризации?**

A: Верно в практическом смысле.

- Автосубтитры YouTube — без диаризации. Сплошной поток текста с таймкодами, без меток спикеров. Иногда вставляют `>>` при смене голоса, но без идентификации.
- Ручные субтитры — бывают, если загрузивший сам разметил (SRT/VTT поддерживают `<v Name>text</v>` или префиксы типа `Имя: ...`). В чужих mock-интервью почти никогда не встречается.
- `youtube-transcript-api` — даже если в VTT есть `<v>`-теги, возвращает только `.text`, `.start`, `.duration`. Диаризации в выходе нет.

Если нужна — отдельно от YouTube:

1. `yt-dlp -x --audio-format mp3 <url>` → аудио.
2. WhisperX (`pip install whisperx`) — Whisper + pyannote, open-source, локально на GPU/CPU.
3. Облачно: Deepgram / AssemblyAI с `diarize=true`. Платно, без GPU.

Из текущего `scripts/transcript_downloader/` это явно вынесено в out-of-scope.

---

**Q: А насколько диаризация нужна для проекта?**

A: Для MVP — не нужна, можно отложить.

Зачем была бы полезна: отчёт должен цитировать кандидата («ты ответил X на вопрос Y»). Без диаризации — риск misattribution (цитата интервьюера уйдёт кандидату).

Почему можно без неё сейчас:

1. В интервью очевидная turn-структура (вопрос → развёрнутый ответ). LLM на Sonnet/Opus справляется с разметкой из контента в ~95% случаев, особенно в behavioral (STAR-ответы однозначно кандидата). Для пилота достаточно.
2. Дедлайн 2026-05-23, скоуп плотный — проект про prompt/agents/eval, а не про ML-препроцессинг.
3. Существующие транскрипты (`transcripts/mock-karpov-*`, `anton-*`) — уже без диаризации, вся baseline-разметка и так на плейн-тексте.

Где реально прижмёт — тогда добавить:

- System Design / диалог-тяжёлые форматы, где и интервьюер, и кандидат долго рассуждают вслух.
- Если метрика «доля корректно атрибутированных цитат» на eval-сете ниже ~90%.

План: не делать до чекпоинта 2026-05-14. Если на eval-сете misattribution >10% — за 1 день прикручивается WhisperX как отдельный шаг (yt-dlp → WhisperX → `transcript.txt` с `[SPEAKER_00] ...` / `[SPEAKER_01] ...`). Аддитивно к `scripts/transcript_downloader/`, сам скрипт не меняется.

Триггер к решению: вручную посмотреть 5–10 интервью на misattribution и принять решение по данным.
