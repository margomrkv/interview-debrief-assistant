## 1) Goal

Сделать ассистента для разбора прошедших интервью.  
**Input:** Job Description + CV + transcript  
**Output:** структурированный отчет (сильные кейсы, проблемные места, рекомендации по улучшению ответов).

## 2) Why this matters

Мы делаем инструмент в первую очередь для себя, так как оба ищем работу:
- Антон: leadership-роли в DS Data Science
- Маргарита: middle Data Analyst (market research) / junior Product Analyst.

Идея проекта от Антона: он записывает интервью через MacWhisper и хочет системно анализировать транскрипты, чтобы улучшать подготовку.

## 4) Current risks / open questions

### Что уже обсудили (краткое резюме)

- **Ценность vs ручной Claude/Cursor:** предварительно договорились, что ценность в стандартизации процесса (фиксированный формат отчета, повторяемый pipeline, измеряемое качество), а не в разовом ad-hoc анализе.
- **Формат результата:** предварительно ориентируемся на `MVP` с фиксированной структурой отчета, а не research-only прототип.
- **Подход к архитектуре:** для v1 приоритет `workflow-first`; агентные функции (долгоживущая персонализация/трекинг) — во 2 фазу.
- **Подход к оценке качества:** нужен baseline + rubric + golden set; это обязательный критерий, подтвержденный на встрече с ментором.

### Что остается открытым (риск)

- [ ] Зафиксировать финальную формулировку ценности в одном абзаце для отчета/демо.
- [ ] Выбрать один домен для v1: DS / Product Analytics / Market Research.
- [ ] Зафиксировать типы интервью для v1 (что в scope, что out of scope).
- [ ] Зафиксировать финальный набор критериев оценки и источник/метод их получения.
- [ ] Подтвердить минимально достаточный объем данных для golden set в рамках сроков.
