# System Design Scoring Notes

Last updated: 2026-05-05

## Что удалось найти в источниках

Есть четкая структура оценки подхода (не только "знание технологий"):

- умение уточнять задачу и ограничения;
- выбор метрик (offline/online) и способ сравнения решений;
- адекватный baseline;
- путь итераций: от простого к более сложному решению;
- реалистичность деплоя/production-подхода.

## Практический критерий для вашей разметки (v0)

0-3 шкала:

- `problem_framing` — качество уточняющих вопросов и постановки задачи;
- `metrics_and_evaluation` — ясность метрик и плана проверки;
- `architecture_tradeoffs` — обоснование решений и компромиссов;
- `execution_plan` — реалистичный путь baseline -> rollout -> iteration.
