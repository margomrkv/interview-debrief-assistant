# Борис: Technical Interview — snapshot

Source: https://btseytlin.github.io/parts/3_interviewing/technical_interview.html  
Captured: 2026-05-05

Ключевые тезисы:

- Типовая структура: опыт -> техвопросы -> задача/кейс -> вопросы кандидата.
- На этапе "опыт" важно не только "что делал", но и "почему так решал".
- Техвопросы обычно типовые и проверяют базовую подготовку.
- Решение задачи/кейса — самый важный этап.
- В алгоритмических задачах важен процесс:
  - уточнение условий;
  - тест-кейсы;
  - рассуждение вслух;
  - проверка решения и кода.
